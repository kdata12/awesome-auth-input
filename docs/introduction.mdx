# awesome-auth-input

A headless, accessible, and fully-typed React component for OTP/auth code inputs with support for numeric, alphabetic, and custom validation patterns.

## API Reference

### AuthCode.Group Props

The root component that manages state and validation for all child inputs.

<ResponseField name="children" type="ReactNode" required>
  The child components, typically `<AuthCode.Input>` components. All inputs must be direct or nested children of the Group.
</ResponseField>

<ResponseField name="name" type="string">
  A string specifying a name for the input control. This name is submitted along
  with the control's value when the form data is submitted.
</ResponseField>

<ResponseField name="validation" type="ValidationPattern" default='{ type: "numeric" }'>
  Specifies what type of validation to apply for each character input.
  
  **Options:**
  - `{ type: "numeric" }` - Numbers only (0-9)
  - `{ type: "alpha" }` - Letters only (a-z, A-Z)  
  - `{ type: "alphanumeric" }` - Letters and numbers
  - `{ type: "custom", regex: RegExp, pattern: string, inputMode: string }` - Custom validation
</ResponseField>

<ResponseField name="autoSubmit" type="boolean" default="true">
  Whether the component should automatically submit when all fields are filled.
  If the field is associated with an HTML `form` element, the form's
  `requestSubmit` method will be called.
</ResponseField>

<ResponseField name="onComplete" type="(value: string) => void">
  Callback fired when all fields are filled. Receives the complete auth code
  value as a string. This fires before auto-submit (if enabled).
</ResponseField>

<ResponseField name="value" type="string">
  The controlled value of the field. When provided, the component operates in controlled mode. The string length must match the number of `<AuthCode.Input>` components.
</ResponseField>

<ResponseField name="onValueChange" type="(value: string) => void">
  Callback fired whenever the field value changes. Use together with `value` to
  create a controlled component.
</ResponseField>

<ResponseField name="defaultValue" type="string">
  The initial value of the uncontrolled field. When provided without `value`,
  the component operates in uncontrolled mode.
</ResponseField>

<ResponseField name="type" type='"text" | "password"' default='"text"'>
  The type of control to render. When set to `"password"`, the input characters
  will be masked with dots.
</ResponseField>

<ResponseField name="disabled" type="boolean" default="false">
  Whether or not the input elements are disabled. When `true`, all inputs will
  be non-interactive.
</ResponseField>

### AuthCode.Input Props

<ResponseField name="index" type="number" required>
  The position of this input (0-based). Must be unique within the group.
</ResponseField>

<ResponseField name="className" type="string">
  CSS class name for styling the input element.
</ResponseField>

<ResponseField name="style" type="React.CSSProperties">
  Inline styles for the input element.
</ResponseField>

<Note>
  `AuthCode.Input` also accepts most standard HTML input attributes like
  `onKeyDown`, `onFocus`, `onBlur`, etc. Props like `value`, `disabled`, and
  `type` are managed by the parent `AuthCode.Group`.
</Note>

## Features

- **Headless** - Bring your own styles, no CSS included
- **Accessible** - Full keyboard navigation, screen reader support, ARIA labels
- **Flexible validation** - Numeric, alphabetic, alphanumeric, or custom regex patterns
- **Controlled & uncontrolled** - Works both ways
- **Auto-submit** - Optional auto-submit when all fields are filled
- **Form integration** - Hidden input for native form submission
- **Password mode** - Mask input characters
- **Paste support** - Paste codes directly
- **TypeScript** - Fully typed with exported types
- **Zero dependencies** - Only React as a peer dependency

## Requirements

<Note>
  This package requires **React 18.0.0 or higher** due to its use of the
  `flushSync` API for synchronous state updates.
</Note>
